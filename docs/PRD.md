## 一、产品概述

### 1.1 产品定位

DeepPrep 是一款**面向全学段学生的 AI 智能备考平台**，核心功能是"拍照搜题 + AI 解析 + 间隔重复复习 + 知识图谱追踪"。

一句话描述：**拍下错题，AI 帮你理解、记住、不再犯错。**

### 1.2 产品目标

| 指标 | 当前现状 | 目标（3个月） |
|------|---------|-------------|
| 核心流程可用性 | ✅ 5 个页面全部可用 | 保持 99%+ 可用率 |
| AI 解析准确率 | 约 85%（依赖模型能力） | 90%+（通过 prompt 优化） |
| 用户留存（周） | 无数据 | 注册用户 7 日留存 > 40% |
| NPS | 无数据 | > 30（及格线） |

### 1.3 目标用户

- **核心用户**：中学生和大学生，有备考需求，习惯用手机/平板拍照
- **场景**：刷题→发现错题→拍照上传→获得解析→定期复习→直到掌握

## 二、用户痛点与解决方案

| 痛点 | 现状 | DeepPrep 方案 |
|------|------|-------------|
| 错题收集效率低 | 手抄到错题本，耗时且容易遗漏 | 📷 拍照 OCR → AI 自动解析，10秒入库 |
| 搜题软件只给答案 | 作业帮/小猿搜题只看答案，不理解过程 | 🤖 AI 逐步解析，适应不同题型（数学推导/英语拆词/代码推演） |
| 错题本堆积不看 | 抄完就忘，没有复习机制 | 🔄 间隔重复算法自动推送该复习的题目 |
| 不知道自己薄弱在哪 | 散落的题目没有结构 | 🧠 自动提取知识点，构建知识图谱，可视化掌握度 |
| AI 出错时无法沟通 | 传统搜题软件是单向的 | 💬 每道题都有 AI 答疑助手，可以追问 |

## 四、功能优先级逻辑

### 为什么选这些功能（MoSCoW 分析）

| 优先级 | 功能 | 理由 |
|--------|------|------|
| 🔴 Must | 智能搜题（OCR + AI 解题） | 核心价值，没有它产品不成立 |
| 🔴 Must | 错题库（保存 + 分组查看） | 闭环第一步，搜题后必须能保存 |
| 🔴 Must | 沉浸式复习（间隔重复 + AI 判卷） | 差异化价值，区别于传统搜题软件 |
| 🟡 Should | 知识图谱 | 展示数据思维，面试加分项 |
| 🟡 Should | 学习统计 | 让用户看到进度，增强留存 |
| 🟡 Should | AI 答疑助手 | 弥补 AI 解析可能出错的缺陷 |
| 🔵 Could | 同类题生成（举一反三） | 锦上添花，提升学习深度 |
| 🔵 Could | 学习日历热力图 | 视觉好看，统计数据已够用 |

### 不做的事情（Won't）

- ❌ 在线刷题题库 —— 资源投入太大，且与"错题管理"定位偏离
- ❌ 社交/排行榜 —— 个人学习工具，不需要社交压力
- ❌ 多语言 —— 目标用户是中文使用者

## 五、复习算法设计

采用 **SM-2 间隔重复变体**（SuperMemo 简化版）：

| 阶段 | 间隔 | 逻辑 |
|------|------|------|
| 答错 | 10 分钟 | 立即重新出现，加深印象 |
| 第1次答对 | 1 天 | 短期记忆巩固 |
| 第2次答对 | 7 天 | 中期记忆转移 |
| 第3次答对 | 15 天 | 长期记忆，可归档 |
| 归档后 | — | 不再推送，知识点标记为"已掌握" |

**设计原则**：保守判卷（默认判错）、渐进间隔、有明确终点（3次答对 = 掌握）。

## 六、技术架构

| 层 | 技术选型 | 选型理由 |
|----|---------|---------|
| 前端 | Streamlit | Python 全栈，一人开发零成本，PC 端体验好 |
| 数据库 | SQLite | 零配置，够用（单用户场景不需要并发） |
| AI | SiliconFlow API (Qwen) | 中文能力强，价格低，API 兼容 OpenAI |
| 部署 | Streamlit Cloud | 免费，GitHub 关联，自动部署 |

## 七、指标体系（AARRR）

| 阶段 | 指标 | 埋点方式 |
|------|------|---------|
| **Acquisition** 获取 | 新增注册用户数 | 数据库 users 表 |
| **Activation** 激活 | 完成首次搜题→保存的用户占比 | questions 表首次保存时间 |
| **Retention** 留存 | 7 日留存率（有没有回来复习） | review_logs 表时间间隔 |
| **Revenue** 收入 | 当前不适用（免费产品） | — |
| **Referral** 推荐 | NPS 评分 | 未来可加反馈入口 |

## 八、已知局限与后续规划

| 局限 | 影响 | 后续计划 |
|------|------|---------|
| 依赖第三方 AI API | API 欠费/限流时产品不可用 | 友好的错误提示（已实现）；增加重试机制 |
| OCR 准确率受图片质量影响 | 模糊/手写/复杂排版识别差 | prompt 持续优化；考虑多模态模型升级 |
| 无移动端适配 | 手机端体验一般（但核心用户会用手机拍照） | 中期用 React Native 重构移动端 |
| 单用户场景 | 无社交/班级功能 | 保持简洁，不做过度扩展 |
