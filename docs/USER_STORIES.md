# DeepPrep 用户故事地图

---

## 用户画像

### 画像 1：中学生 小李

> "每天刷题，错题本抄了一本又一本，但从来不翻。"

**背景**：16岁，高中生。每周做 50+ 道数学题，错题传统方法是手抄到错题本，但抄完就没再看过。考试前才发现同样类型的题还是不会。

**核心需求**：快速记录错题 → 理解为什么错 → 定期复习 → 不再犯同类错误

**使用场景**：
1. 晚自习做完一套数学卷子，拍照上传 3 道错题
2. 周末打开 DeepPrep，系统提醒他今天有题目该复习了
3. 复习后 AI 判卷告诉他哪里还没理解透
4. 知识图谱显示"二次函数"知识点还是黄色（部分掌握），他去错题库找到相关题目再次练习

---

### 画像 2：大学生 小王

> "专业课没有标准答案，刷题全靠自己总结。"

**背景**：21岁，计算机专业大三。在准备秋招，刷 LeetCode 和专业课题目。错题散落在各个笔记 App 里，没有体系化的复习。

**核心需求**：跨学科管理错题 → AI 能理解编程题/专业课题目 → 算法推演式的解析

**使用场景**：
1. 刷到一道 LeetCode 哈希表题目不会做，截图上传
2. AI 给出算法思路 → 关键步骤推演 → 时间复杂度的解析
3. 标记为错题，归入"专业课(理工)"学科
4. 复习时看到同类题推荐，举一反三

---

### 画像 3：考研人 小张

> "英语阅读理解错了一堆，但每道题的错因都不一样。"

**背景**：23岁，大四考研。英语阅读理解是弱项，同样是选错，有的是因为单词不认识，有的是理解偏差，有的是推理过度。

**核心需求**：精细化错因分析 → 逐词拆解的解析 → 知识点级别的掌握度追踪

**使用场景**：
1. 做完一套英语真题，拍下做错的阅读理解
2. AI 逐词拆解关键单词 → 语法结构分析 → 翻译要点
3. 知识点标注为"阅读理解/推理判断"
4. 知识图谱显示英语科目下有 5 个待掌握知识点，逐一攻克

## 用户旅程地图

### 核心旅程：从错题到掌握

```
阶段           用户动作              情绪曲线      触点
──────────────────────────────────────────────────────────
发现错题      做完题，发现自己错了    😟 沮丧       拍照上传
    ↓
AI 解析       看到详细的解题步骤      😯 原来如此    AI 解析页
    ↓
保存入库      一键保存到错题库        😊 省事        "存入题库"按钮
    ↓
复习提醒      收到今日待复习通知      😅 该复习了    首页 Dashboard
    ↓
复习答题      回忆+写出答案           🤔 努力回想    沉浸式复习页
    ↓
AI 判卷       看到对勾或修改建议      😤/😊        判卷反馈
    ↓
反复巩固      2-3 次答对后            🎉 终于会了   知识图谱绿色
    ↓
归档          知识点标记"已掌握"      🏆 成就感     知识图谱
```

### 痛点时刻与解决方案

| 痛点时刻 | 负面情绪 | 解决方案 |
|----------|---------|---------|
| 拍照后要等很久才有结果 | 😤 不耐烦 | 流式输出，实时显示解题步骤 |
| AI 解析是错的但没法纠正 | 😠 失望 | "报告答案有误"按钮 + AI 答疑 |
| 打开 App 不知道该干嘛 | 😕 迷茫 | 首页 Dashboard 显示今日任务 |
| 复习时忘了题目是什么 | 🤯 困惑 | 题目+解析完整展示，文本优先 |
| 同一个题型反复错 | 😩 绝望 | 同类题生成 + 知识点追踪 |
